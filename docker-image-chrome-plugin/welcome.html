Docker Hub Helper

从 tags 页直接拉镜像，不再手抄命令。

这个扩展做的事情很直接：在 Docker Hub 的 tags 页面找到正确架构，点一下按钮，生成可导入的 tar 包。 出问题时，再去弹出层看进度、错误和历史记录。

示例 tags 页面 打开任务弹出层
公益项目
临时救急优先，配置负担尽量低。

目标是帮助开发和运维人员在没有完整开发环境时，先把镜像取下来、导进去、把问题处理掉。

节点贡献说明

如果后续有广告收益，可以按实际流量向代理节点提供者支付一定费用。代理服务本身资源占用很低，主要消耗带宽和少量缓存磁盘。

如何使用插件

01
1. 打开镜像的 tags 页面

例如 `busybox`、`python` 或你自己的镜像仓库。在每个架构行右侧会出现下载按钮。

02
2. 选对 tag 和架构再下载

同一个 tag 可能有多个架构。先确认是 `amd64`、`arm64`、`arm/v7` 还是其它变体，再点击对应按钮。

03
3. 在弹出层查看任务状态

点击浏览器工具栏里的扩展图标，可以看到活动任务、下载历史、错误信息，也可以取消进行中的任务。

04
4. 下载完成后直接导入

插件输出的是 tar 包，下载完成后可以用 `docker load -i image.tar` 或 `ctr` / `nerdctl` 相关命令直接导入。

认证
只有私有镜像或受限仓库时才需要填 Docker Hub 账号或令牌。
代理
只有本地无法稳定拉取，或者你要给更多人提供下载能力时，才需要代理节点。
结果
下载结果是 tar 包，不是镜像 ID。拿到文件后直接导入运行环境即可。

可选：部署代理节点

如果你有公网 IP 云服务器，并愿意贡献一个轻量代理节点，可以直接执行下面的安装命令。插件本身不依赖你部署节点才能使用。

命令说明： `<YOUR_PUBLIC_IP>` 替换成你的云服务器公网 IP。脚本运行后会在安装依赖前要求确认， 然后用 pm2 管理代理服务进程。

公网部署前先放行云服务器的 `7001/tcp`。记录服务会反向校验 `http://<YOUR_PUBLIC_IP>:7001/health`，端口没开时注册会超时。

如果你的云服务器访问 Docker Hub 仍然需要上游代理，可以额外在执行时加上： `USE_PROXY=true PROXY_URL=http://127.0.0.1:7890`

代理节点是可选能力，不影响插件基本使用。服务本身资源占用低，主要消耗出口带宽和少量磁盘缓存。
