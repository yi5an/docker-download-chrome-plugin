插件已经装好，先跑通下载，再考虑扩容代理。

这个扩展的核心路径很简单：在 Docker Hub 的 tags 页面点下载按钮，打开弹出层查看进度， 需要时补 Docker Hub 认证。你如果准备长期使用，建议把代理服务部署到一台有公网 IP 的云服务器上。

打开示例 tags 页面 打开任务弹出层

为什么推荐公网云服务器

本机代理只适合你自己调试。放到公网云服务器上以后，节点更稳定，带宽更可控，也更容易被记录服务选中。

  • 公网 IP 让插件和代理节点之间不再依赖本机网络环境
  • 带宽和可用性通常比家宽更稳定
  • 后续可以逐步扩成多节点代理池

如何使用插件

1. 打开 Docker Hub 的 tags 页面

例如 `busybox` 或你自己的镜像仓库，在每个架构行右侧都会出现下载按钮。

2. 点击目标架构的下载按钮

插件会自动创建下载任务。大镜像会先拉 layer，再打包成 tar 文件。

3. 打开弹出层看进度

点击浏览器工具栏里的扩展图标，就能看到活动任务、历史记录和错误信息。

4. 私有镜像再配置 Docker Hub 认证

只有下载私有镜像或受限仓库时才需要在弹出层里保存用户名 / 令牌。
